포트폴리오 관리 — 보유 종목을 구성·추가(매수)·삭제(매도)하고 평가합니다.

요청: $ARGUMENTS

---

## 역할

사용자의 보유 포트폴리오(`data/portfolio.md`)를 관리합니다. 종목을 입력받아 **간단한 종목 정보를 보여주고** 보유 정보(수량·매입가·통화·매입시환율)를 입력하도록 도와 테이블에 반영합니다. 모든 편집은 아래 CLI를 통해 수행합니다(직접 마크다운을 손으로 고치지 말 것 — 도구가 주변 텍스트를 보존하며 안전하게 편집).

> 종목별 심층 분석이 필요하면 `/invest:stock <TICKER>`, 처음 세팅은 `/invest:setup`.

## 모드 판정 (가장 먼저)

`$ARGUMENTS`(없으면 사용자에게 질문)로 의도를 판정합니다.

| 의도 | 키워드 예시 | 모드 |
|------|------------|------|
| 매수/추가 | "샀어", "매수", "추가", "buy", "add" + 티커 | **B. 추가** |
| 매도/삭제 | "팔았어", "매도", "삭제", "제거", "sell", "remove" + 티커 | **C. 삭제** |
| 일괄 구성 | "구성", "세팅", "포트폴리오 만들어", 티커 목록 다수 | **A. 일괄구성** |
| 평가/현황 | "현황", "평가", "얼마", 인자 없음 | **D. 평가** |

판정 결과를 한 줄로 알린 뒤 진행합니다. 애매하면 사용자에게 확인합니다.

---

## 공통: 종목 정보 표시 + 입력 받기

종목을 추가/구성할 때 **각 티커마다** 먼저 정보를 조회해 보여줍니다.

```bash
python3 src/tools/cli.py portfolio quote <TICKER>
```

표시 항목(있는 것만): **종목명 · 현재가 · 통화 · 섹터/업종 · 52주 범위 · 시총**. 그런 다음 아래 4가지를 입력받습니다.

| 입력 | 설명 |
|------|------|
| **수량** | 보유 주식 수 |
| **매입가** | 1주당 평균 매입 단가(해당 종목 통화 기준) |
| **통화** | `USD` 또는 `KRW` — 미입력 시 티커로 자동 추정(`.KS`/`.KQ`→KRW, 그 외 USD). `quote`의 `suggested_currency`를 기본값으로 제시 |
| **매입시환율** | **USD 종목만** — 매입 시점 원/달러 환율(예 `1490`). 모르면 생략 가능하나 **원화 손익이 부정확**해진다고 안내(현지통화 USD 수익률은 정확). KRW 종목은 불필요 |

입력을 모았으면 `add`로 반영합니다.

```bash
python3 src/tools/cli.py portfolio add <TICKER> --qty <수량> --price <매입가> [--ccy USD|KRW] [--fx <매입시환율>]
```

- 같은 티커가 이미 있으면 **덮어쓰기(update)** — 평단 갱신/추가매수 반영에 사용.
- `warnings`(예: USD인데 환율 누락)는 사용자에게 전달합니다.

## 공통: 연동 파일 동기화 (필수)

`portfolio.md` 를 바꾸면 **반드시** 아래 두 파일도 동기화합니다(둘 다 `.gitignore` — 로컬 전용, Edit/Write로 직접 편집).

- **`data/theses.md`** — 보유 종목 투자 Thesis:
  - 매수(신규): `### <TICKER> — (회사명/테마)` 섹션을 추가합니다. Thesis·핵심 가정(KPI)·🟡흔들림·🔴훼손 트리거를 채웁니다(사용자에게 논거를 묻거나, 모르면 `/invest:stock`으로 분석 후 채울 것을 제안).
  - **매도(전량): 해당 `### <TICKER>` 섹션을 제거합니다.**
- **`data/positions.md`** — 포지션 비중:
  - `python3 src/tools/cli.py portfolio --json` 으로 갱신된 비중을 받아 순위 표·기준일(`USDKRW`)·집중도 메모(IREN 실효노출, 상위2 합산)를 다시 씁니다.
  - 매도/매수 사실을 한 줄 메모로 남깁니다(예: "YYYY-MM-DD SPCX 전량 매도 → 현금").

## 모드 A: 일괄구성

1. 사용자가 준 종목 목록을 파싱합니다(콤마/공백/줄바꿈 구분). 종목만 주고 보유정보가 없으면, 종목별로 `quote`를 보여주며 수량·매입가·통화·환율을 차례로 입력받습니다(여러 종목이면 한 번에 표로 받아도 됨).
2. 각 종목을 `portfolio add` 로 추가합니다.
3. 완료 후 **D. 평가**를 실행해 현재 평가/손익/비중 테이블을 보여줍니다.

> 깨끗한 새 구성: `data/portfolio.md`가 없으면 첫 `add`에서 자동 생성됩니다(예시 행 없이 시작). 기존 파일에 불필요한 행이 있으면 `remove`로 정리합니다.

## 모드 B: 추가 (매수)

1. 티커로 `quote`를 보여줍니다.
2. 수량·매입가·통화·매입시환율(USD)을 입력받습니다(일부만 줬으면 나머지만 질문).
3. `portfolio add` 실행 → 결과(action: added/updated)와 warnings 보고.
4. **연동 동기화**(필수): `theses.md`에 Thesis 섹션 추가 + `positions.md` 비중 재산출. (위 *공통: 연동 파일 동기화* 참조)
5. 이어서 평가 테이블을 보여줄지 물어봅니다(또는 바로 D 실행).

## 모드 C: 삭제 (매도)

1. 대상 티커를 확인합니다(전량 매도 전제 — 부분 매도면 B의 update로 수량 조정).

```bash
python3 src/tools/cli.py portfolio remove <TICKER>
```

2. 결과 보고: `removed`(제거됨) / `not_found`(보유에 없음).
3. **연동 동기화**(필수): `theses.md`에서 해당 `### <TICKER>` 섹션 제거 + `positions.md` 비중 재산출·매도 메모. (위 *공통: 연동 파일 동기화* 참조)
4. 매도 후 남은 포트폴리오를 **D. 평가**로 보여줍니다(집중도 변화 강조).
5. **자금 재배치 추천 (필수 안내)**: 매도로 생긴 현금을 어디에 넣을지 고민하도록, **`/invest:market`** 으로 지금 자금이 몰리는 핫섹터·테마를 발굴해 후보를 찾을 것을 추천합니다.
   - 사용자가 원하면 `/invest:market "<관심 테마 또는 '지금 뜨는 섹터'>"` 로 바로 이어갑니다.
   - 집중도가 높은 상태(상위2 합산 50%+ 등)면, 재배치 시 **집중 분산**도 함께 고려하도록 짚어줍니다.

## 모드 D: 평가

```bash
python3 src/tools/cli.py portfolio
```

- 현재가·환율을 조회해 종목별 평가금액·손익(현지통화/원화)·비중과 합계 테이블을 출력합니다.
- 환율을 수동 지정하려면 `--fx <환율>`.
- 핵심 요약(총평가액·총손익·최대 비중/집중 종목)과 `warnings`를 함께 정리해 보고합니다.

## 마무리

- 보유가 바뀌었으면 `theses.md`·`positions.md` 동기화가 끝났는지 확인합니다(위 *공통: 연동 파일 동기화* — 매수/매도 시 필수).
- 매도 후라면 자금 재배치 후보 발굴을 위해 `/invest:market` 을 안내했는지 확인합니다.
- 이 데이터 파일들은 `.gitignore` 대상이라 로컬에만 보관됩니다.

> ※ 모든 평가·정보는 참고용이며 투자 권유가 아닙니다.
