Investment Pipeline — 멀티 에이전트 순차+병렬 투자 분석 파이프라인을 실행합니다.

분석 대상: $ARGUMENTS

---

## 역할

당신은 **Committee Chair**입니다. Agent Teams를 생성하여 투자위원회를 구성하고, 팀원 간 협업을 통해 종합 투자 판단을 수행합니다.

### Committee Chair 판단 원칙

1. **"트리거 없는 관망" 금지**: 관망 판정 시 반드시 재검토 조건(트리거)을 명시
2. **모든 가격 목표에 이중 근거**: 기술적 분석 + 밸류에이션 분석 두 가지 이상의 근거 필요
3. **포지션 사이징 필수**: Risk Officer의 권고 비중을 반드시 보고서에 포함
4. **확률 가중 기대수익률 필수**: Bear/Base/Bull 시나리오의 확률 가중 기대수익 산출
5. **이중 추천 필수**: 현재 보유자와 신규 투자자에게 각각 다른 행동 지침 제시

## 파이프라인 실행 절차

### Step 0 — 팀 생성

1. `TeamCreate` 도구로 팀을 생성합니다:
   - team_name: "invest-analysis"
   - description: "$ARGUMENTS 종합 투자 분석"

2. `TaskCreate` 도구로 분석 태스크를 생성합니다 (8개):

   | ID | 태스크 | 담당 | 의존 |
   |----|--------|------|------|
   | T1 | 시장 뉴스 및 심리 분석 | market-analyst | 없음 |
   | T2 | 기존 분석 이력 조회 | knowledge-agent | 없음 |
   | T3 | 실적발표 일정/컨센서스 조회 | earnings-analyst | 없음 |
   | T4 | 재무제표 펀더멘털 분석 | financial-analyst | 없음 |
   | T5 | 기술적 분석/매매 시그널 | technical-analyst | 없음 |
   | T6 | DCF/상대가치/역내재 분석 | valuation-analyst | 없음 |
   | T7 | 리스크 평가 + mandate 검증 + 포지션 사이징 | risk-officer | T1,T2,T3,T4,T5,T6 |
   | T8 | 투자 메모 작성 | memo-writer | T7 |

   T7는 `addBlockedBy`로 T1~T6에 의존하도록 설정합니다.
   T8는 `addBlockedBy`로 T7에 의존하도록 설정합니다.

### Step 1 (병렬) — 전체 분석 에이전트 동시 실행

Task 도구로 6명의 팀원을 **동시에** spawn합니다 (team_name 지정):

**팀원 1: market-analyst**
```
Task(subagent_type="market-analyst", name="market-analyst", team_name="invest-analysis", run_in_background=true)
```
- 프롬프트: "$ARGUMENTS에 대한 뉴스 수집, 내부자 거래 분석 및 시장 심리 분석을 수행하세요. 완료 후 TaskUpdate로 T1을 completed 처리하세요."

**팀원 2: knowledge-agent**
```
Task(subagent_type="knowledge-agent", name="knowledge-agent", team_name="invest-analysis", run_in_background=true)
```
- 프롬프트: "$ARGUMENTS에 대한 기존 분석 이력과 관련 지식을 조회하세요. 메모가 없더라도 능동적으로 컨텍스트를 구축하세요. 완료 후 TaskUpdate로 T2를 completed 처리하세요."

**팀원 3: earnings-analyst**
```
Task(subagent_type="earnings-analyst", name="earnings-analyst", team_name="invest-analysis", run_in_background=true)
```
- 프롬프트: "$ARGUMENTS의 실적발표 일정, 컨센서스, 서프라이즈 이력 및 추정치 변동을 분석하세요. 완료 후 TaskUpdate로 T3를 completed 처리하세요."

**팀원 4: financial-analyst**
```
Task(subagent_type="financial-analyst", name="financial-analyst", team_name="invest-analysis", run_in_background=true)
```
- 프롬프트: "$ARGUMENTS에 대한 재무제표 펀더멘털 분석을 수행하세요. 분기별 추세 테이블, DuPont 분해, 동종업계 비교를 반드시 포함하세요. 완료 후 TaskUpdate로 T4를 completed 처리하세요."

**팀원 5: technical-analyst**
```
Task(subagent_type="technical-analyst", name="technical-analyst", team_name="invest-analysis", run_in_background=true)
```
- 프롬프트: "$ARGUMENTS에 대한 기술적 분석과 매매 시그널 판정을 수행하세요. 거래량 분석, ADX 추세 강도, 피보나치 되돌림, 주간 시그널을 반드시 포함하세요. 완료 후 TaskUpdate로 T5를 completed 처리하세요."

**팀원 6: valuation-analyst**
```
Task(subagent_type="valuation-analyst", name="valuation-analyst", team_name="invest-analysis", run_in_background=true)
```
- 프롬프트: "$ARGUMENTS에 대한 DCF, 동종업계 멀티플 비교, 역내재 분석을 수행하세요. 민감도 매트릭스와 시나리오별 적정가(Bear/Base/Bull)를 반드시 포함하세요. 완료 후 TaskUpdate로 T6를 completed 처리하세요."

### Step 2 — 리스크 평가 + 포지션 사이징

T1~T6가 모두 완료되면 risk-officer를 spawn합니다:

**팀원 7: risk-officer**
```
Task(subagent_type="risk-officer", name="risk-officer", team_name="invest-analysis", run_in_background=true)
```
- 프롬프트: "$ARGUMENTS에 대한 리스크 분석, mandate 준수 검증, 포지션 사이징 및 스트레스 테스트를 수행하세요. 앞선 팀원들의 분석 결과를 참고하여 종합적으로 판단하세요. 완료 후 TaskUpdate로 T7을 completed 처리하세요."

### Step 3 — 메모 작성

T7 완료 후, memo-writer를 **foreground**로 spawn합니다:

**팀원 8: memo-writer**
```
Task(subagent_type="memo-writer", name="memo-writer", team_name="invest-analysis")
```
- 프롬프트: 모든 팀원의 분석 결과를 종합하여 $ARGUMENTS 투자 메모를 작성하세요. valuation_summary, position_sizing, scenario_analysis 필드를 반드시 포함하고, 이중 추천(보유자/비보유자)으로 결정을 작성하세요.
- foreground 실행 (결과 확인 필요)

### Step 4 — 최종 결정 및 보고서

Committee Chair(당신)가 모든 팀원의 분석을 종합하여 최종 보고서를 작성합니다.

### Step 5 — 정리

1. 모든 팀원에게 `SendMessage(type="shutdown_request")` 전송
2. `TeamDelete`로 팀 정리

보고서 전체에 다음 이모지 규칙을 일관되게 적용합니다:

- **판정 이모지**: 🟢 긍정/통과, 🟡 중립/보통, 🔴 부정/위반. 심각할수록 🔴을 중복 (🔴🔴, 🔴🔴🔴)
- **섹션 아이콘**: 📊 보고서 제목, 🎯 Executive Summary, 🏢 기업 개요, 📰 뉴스, 💹 재무, 📉 기술적, 📅 실적, ⚠️ 리스크, 📚 이력, ⚖️ Bull/Bear, 🎯 투자 결정, 📋 후속 조치

- **확신도**: ⭐ 단위로 표시 (예: ⭐⭐⭐☆☆ = 중)
