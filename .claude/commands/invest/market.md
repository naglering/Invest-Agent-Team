시장분석 — 섹터/테마/매크로 환경을 분석하고 투자 전략을 수립합니다.

분석 대상: $ARGUMENTS

---

## 역할

당신은 **Committee Chair**입니다. 특정 개별 종목이 아닌 **섹터·테마·매크로 환경**을 분석합니다. 시장 전반의 흐름을 읽고, 그 안에서 유망한 종목군을 식별하여 투자 전략을 제시합니다.

> 단일/복수 개별 종목의 심층 분석이 필요하면 `/invest:stock <TICKER>`로 안내합니다.

## 분석 대상 유형

| 유형 | 예시 |
|------|------|
| 🏭 섹터 | "반도체 섹터 전망", "방산주 어때?" |
| 🎯 테마 | "AI 인프라 테마", "원자력 르네상스" |
| 🌍 매크로 | "금리 인하기 성장주 전략", "달러 약세 수혜주" |

## 실행 절차

### 0단계: 시장 스캔 → 핫섹터 랭킹 (능동 발굴)

분석에 들어가기 전 **먼저 자금이 어디로 도는지** 스캔합니다. 페르소나는 자금이 몰리는 메가트렌드를 먼저 발굴해 올라타는 투자자입니다 — 사용자가 섹터를 지목하지 않아도 능동적으로 후보를 제시합니다.

```bash
python3 src/tools/cli.py sectors
```

이 도구는 메가트렌드 테마 + 전통 섹터 ETF의 1/3/6M 수익률·SPY 대비 상대강도·50/200일선 위치를 `momentum_score`로 랭킹합니다. 출력 키: `ranking`, `top_megatrend`, `rating`(🟢자금유입/🟡중립/🔴자금이탈), `rating_rule`, `spy_3m_return_pct`.

스캔 결과로 분기 처리합니다:

- **특정 섹터/테마/매크로 주제가 지정됨** → **모드 S (지정 섹터 심층 분석)**: 해당 항목의 `momentum_score`·SPY 대비 상대강도·자금흐름 `rating`을 **먼저 보여준 뒤** 아래 1~4단계 심층 분석을 진행합니다. (지정된 대상이 실제로 자금이 빠지는 섹터라면 그 사실을 명시.)

- **섹터 미지정 / 공백 / 모호함** → **사용자에게 방향을 질문**합니다. `AskUserQuestion`으로 다음 두 옵션을 제시하고, 선택에 따라 분기합니다 (스캔 결과를 질문 맥락에 요약해 함께 보여줌):
  1. **전반적인 시장 상황 분석** → **모드 M** (아래 "모드 M" 절차). 특정 섹터에 매이지 않고 매크로·자금흐름·섹터 로테이션·시장 국면을 종합한 시장 전반 리포트.
  2. **후보 제시 (핫섹터 → 선택)** → `top_megatrend`와 상위 랭킹 테마를 후보로 제시하고, 사용자가 고른 테마/섹터로 **모드 S** 심층 분석을 진행. (사용자가 즉답하지 않으면 모멘텀 1위 테마를 기본 추천.)

> 이 단계는 기존의 "사용자가 섹터를 지목해야만 작동하는 수동 구조"에 **능동 발굴 + 방향 선택**을 더한 것입니다. 사용자가 처음부터 명시적으로 "전반 분석"/"시장 전체" 등을 요청했다면 질문을 생략하고 바로 모드 M으로 갑니다.

---

## 모드 M: 전반적인 시장 상황 분석 (섹터 미지정 시)

특정 섹터가 아니라 **시장 전체의 국면**을 읽고, 자금이 어디서 어디로 도는지(로테이션)와 관전 포인트를 제시합니다. 개별 섹터 심층(1~4단계)을 모든 섹터에 돌리지 않고, 0단계 `sectors` 스캔 + 매크로를 종합합니다.

### M-1. 데이터 수집 (병렬)

- 0단계 `cli.py sectors` 결과(`ranking` 전체, `top_megatrend`, `spy_3m_return_pct`)를 그대로 활용합니다.
- **market-analyst (필수)**: "현재 **시장 전반**의 매크로 환경을 분석하세요. WebSearch로 (a) 금리/연준 정책 방향과 유동성, (b) 인플레이션·고용 등 핵심 지표, (c) 달러·국채금리·신용스프레드, (d) 시장 심리(VIX, 위험선호/회피), (e) 주요 정책/지정학 이벤트를 검색하세요. **검색어 연도는 하드코딩하지 말고 현재 연도/분기를 동적으로 사용.**"
- **earnings-analyst (선택)**: 어닝 시즌이면 "시장 전반 실적 모멘텀(전체 추정치 리비전 방향, 어닝 시즌 진행도)"을 요약.

### M-2. 시장 국면 종합

- **자금흐름 폭(breadth)**: `ranking`에서 🟢자금유입 / 🟡중립 / 🔴자금이탈 테마·섹터 수를 집계해 시장이 위험선호(risk-on)인지 회피(risk-off)인지, 자금이 광범위 확산인지 소수 테마 쏠림인지 판단.
- **섹터 로테이션 맵**: 어디서 돈이 빠져(🔴) 어디로 들어오는지(🟢) 흐름을 서술. SPY 3M(`spy_3m_return_pct`) 대비 초과/미달 테마.
- **톱 메가트렌드 관전 포인트**: `top_megatrend` 상위 2~3개를 골라 각각 왜 자금이 몰리는지·서사·진입 국면(초기/확산/과열)을 1~2줄로.

### M-3. 보고서 (모드 M 전용 형식)

아래 "보고서 형식"의 **🔥 시장 자금흐름 랭킹 + 🌍 매크로/시장 환경 + ⚖️ 시장 Bull/Bear + 🎯 투자 전략**을 사용하되, 대상은 "시장 전체"입니다. 추가로:

- **🧭 시장 국면 한 줄**: risk-on/off, 자금 광범위/쏠림, 추천 자세(공격/중립/방어).
- **🔁 섹터 로테이션 맵**: 🟢유입 테마 ⬅ 🔴이탈 테마 흐름 표.
- **👀 관전 리스트(Watchlist)**: 자금유입 상위 테마별 대표 종목 2~3개(`cli.py momentum`으로 RS/돌파 확인)와 "지금 추세 진입/눌림 대기" 태그.
- 마무리에 "특정 테마를 더 깊게 보려면 `/invest:market <테마>`로 모드 S 심층 분석" 안내.
- 마무리에 "순수 매크로·크로스에셋·레짐·자산배분 틸트(시장 전체의 날씨)를 더 깊게 보려면 `/invest:macro`(톱다운 레짐 리포트)" 안내. (모드 M이 '어디로 돈이 도는가'라면 macro는 '어떤 경기·정책·유동성 레짐인가'.)

> 모드 M은 깊이보다 **시야(어디에 베팅할지 후보 좁히기)**가 목적입니다. 개별 종목·단일 섹터 심층은 모드 S 또는 `/invest:stock`으로 위임.

## 모드 S: 지정 섹터/테마 심층 분석 (1~4단계)

> 섹터/테마/매크로 주제가 지정됐거나(0단계), 모드 M의 후보 제시에서 사용자가 특정 테마를 고른 경우 이 경로로 진행합니다.

### 1단계: 질의 파싱

$ARGUMENTS에서 다음을 추출합니다:
- **주제**: 섹터/테마/매크로 키워드
- **관련 주요 종목 후보**: WebSearch로 해당 섹터/테마의 대표 종목 3~7개를 식별 (예: "AI 반도체" → NVDA, AMD, TSM, AVGO 등)
- **분석 깊이**: 빠른 개관 vs 종목 스캔 포함 심층 (기본: 심층)

> WebSearch 검색어에 연도를 넣을 때는 **연도를 하드코딩하지 말고 현재 날짜(올해/이번 분기)를 동적으로 반영**합니다 (예: "<섹터> outlook <현재연도>"). 최신성이 떨어지면 직전 연도까지 보조 검색.

### 2단계: 병렬 분석 실행

Task 도구로 아래 에이전트를 **background 병렬**로 동시에 실행합니다:

**market-analyst (필수)**
```
Task(subagent_type="market-analyst", run_in_background=true)
```
- 프롬프트: "'$ARGUMENTS' 섹터/테마에 대해 분석하세요. WebSearch로 (a) 섹터 전반 뉴스·내러티브, (b) 관련 ETF 자금 유출입과 섹터 모멘텀, (c) 금리·환율 등 매크로 환경의 영향, (d) 정부 정책/규제 방향, (e) 경쟁 구도와 시장 점유 변화를 검색하세요. 섹터를 대표하는 주요 종목 3~7개를 식별하고 각각의 최근 동향을 요약하세요. **검색어에 연도를 넣을 경우 하드코딩하지 말고 현재 연도/분기를 동적으로 사용하세요.**"

**knowledge-agent (필수)**
```
Task(subagent_type="knowledge-agent", run_in_background=true)
```
- 프롬프트: "data/histories에서 '$ARGUMENTS' 섹터/테마와 관련된 기존 분석 이력을 조회하세요. 관련 종목의 과거 메모, 이전 투자 판단, 당시 대비 변화 포인트를 정리하세요."

**earnings-analyst (필수)**
```
Task(subagent_type="earnings-analyst", run_in_background=true)
```
- 프롬프트: "'$ARGUMENTS' 섹터 주요 종목들의 실적발표 일정과 컨센서스를 조회하세요. 섹터 전반의 실적 모멘텀(상향/하향 추세)을 판단하세요. **특히 주요 종목별 EPS/매출 추정치 리비전(상향/하향) 방향과 폭을 집계해 '섹터 추정치 리비전 모멘텀'을 산출하세요** — 추정치가 일제히 상향되는 섹터는 자금 쏠림의 선행 촉매이므로 별도로 강조합니다."

### 3단계: 주요 종목 경량 스캔 (심층 모드)

1단계에서 식별한 대표 종목 3~5개에 대해 **financial-analyst**를 background 병렬로 실행하여 경량 펀더멘털을 수집합니다 (개별 풀 분석이 아닌 비교용 핵심 지표만):

```
각 대표 종목마다:
  Task(subagent_type="financial-analyst", run_in_background=true,
       prompt="<TICKER> 재무 핵심 지표(밸류에이션, 성장성, 수익성)를 간결히 요약하세요. 섹터 비교용입니다.")
```

동시에 각 종목의 **모멘텀/상대강도**를 수집합니다 — `momentum` 도구로 종목 RS·52주 신고가 돌파 셋업·거래량 급증·`momentum_score`를 확보합니다:

```bash
python3 src/tools/cli.py momentum <TICKER>
```

펀더멘털(financial-analyst)과 모멘텀(momentum 도구)을 결합해 4단계 종목 스캔 표를 채웁니다.

### 4단계: 종합 → 섹터 투자 전략

모든 에이전트 결과를 종합하여 섹터 내 상대 매력도를 평가하고, 톱픽과 비중 전략을 도출합니다.

## 저장 + PDF (양 모드 공통, 필수)

작성한 보고서 **전문**을 `data/histories/`에 저장합니다(PDF는 사용자에게 물어본 뒤 생성). 시장분석은 개별 종목이 아니므로 의사(疑似)티커를 사용합니다 — **모드 M(시장 전반)이면 `SECTOR-REVIEW`, 모드 S(지정 섹터/테마)면 섹터 약칭 대문자**(예: `ENERGY`, `SEMI`, `URANIUM`).

```bash
# (1) 종합보고서 저장 → data/histories/<YYYY-MM-DD>_<PSEUDO>/report.md
python3 src/tools/cli.py memo report <PSEUDO>   # stdin으로 보고서 전문 입력
```

- 시장분석 보고서는 **요약(summary.md) 없이 report.md만** 저장합니다(memo-writer 파이프라인 비해당). 따라서 PDF는 항상 `--mode deep`(report.md 기반).
- **(2) PDF 출력 여부 질문 (필수)**: 저장 후 사용자에게 보고서를 그대로 출력하고 `report.md` 경로를 한 줄 보고한 뒤, **`AskUserQuestion`으로 PDF 생성 여부를 묻습니다**(자동 생성하지 않음). 생성 선택 시에만 아래를 실행하고 `report.pdf` 경로를 보고합니다. 미선택 시 종료.
  ```bash
  python3 src/tools/cli.py report-pdf <PSEUDO> --meta-stdin --mode deep   # stdin으로 메타 JSON 입력
  ```
- **메타 JSON 구성**: `company`(예: "에너지 섹터 분석" 또는 "시장 자금흐름 리뷰"), `ticker`(의사티커), `report_type`("Sector/Market"), `date`, `headline`(핵심 판단 1줄), `key_data`(시장 국면·추천 자세·상위 유입 테마·핵심 지표 등 라벨/값 배열), `takeaways`(3-4개). **개별 종목 목표가가 없으면 `scenarios`는 생략**(Risk-Reward 차트 자동 생략).

## 보고서 형식

### 시각화 규칙

- **판정 이모지**: 🟢 긍정/매력, 🟡 중립, 🔴 부정/회피
- **방향**: ⬆️ 긍정 영향, ⬇️ 부정 영향, ➖ 중립
- **확신도**: ⭐ 단위 (예: ⭐⭐⭐☆☆)
- **물결표(`~`) 금지**: 마크다운 뷰어(GFM)는 `~`를 취소선으로 해석하여 한 줄에 둘 이상 쓰면 그 사이가 취소선 처리된다. 숫자 **범위**는 `–`(en dash)나 `-`로(예: `$360–430`, `+30–40%`, `15–25%`), **근사값**은 `약`으로(예: `약 95%`) 표기한다. `~`를 직접 쓰지 않는다.
- **용어 한글화 (필수)**: 보고서 작성 전 `.claude/commands/invest/glossary.md`를 Read해 규칙을 적용한다. 영어를 한글 문장에 그대로 박지 말고(rotation → 순환매, breadth → 자금흐름 폭, risk-on/off → 위험선호/회피), 음차는 표준어 우선(로테이션 → 순환매).

**🏆 섹터 내 톱픽:**

> 톱픽 선정 가중치: **펀더멘털 40% + 모멘텀/RS 35% + 촉매/서사 25%**. 각 종목을 세 축으로 점수화해 합산 순위를 매깁니다.

| 순위 | 종목 | 선정 이유 | 종합점수(펀40/모35/서25) | 진입 적합성 |
|------|------|----------|--------------------------|-------------|
| 1 | TICKER | 2~3줄 근거 | 예: 펀 32 / 모 30 / 서 20 → 82 | 추세 진입 / 눌림 대기 / 과열 회피 |
| 2 | TICKER | | | 추세 진입 / 눌림 대기 / 과열 회피 |

- **진입 적합성 3분류**:
  - 🟢 **추세 진입(돌파)**: RS 강세 + 52주 신고가 돌파/근접 + 거래량 동반 → 돌파 시 초기 진입.
  - 🟡 **눌림 대기**: 추세 양호하나 단기 과열/이격 → 50일선·전고점 등 지지선 눌림 대기.
  - 🔴 **과열 회피**: 52주 고점 대비 과도한 단기 급등·이격 확대 → 신규 진입 보류.
- **피라미딩 가이드**: 추세 진입 종목은 분할 진입 후 가설이 맞을 때만 불타기(추가 매수)로 비중 확대 — 손절선을 진입가/직전 지지선 위로 올리며 평균단가가 아닌 추세로 키운다. 추세 훼손(손절선 이탈) 시 추가 매수 중단.
